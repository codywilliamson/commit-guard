# Contributing

Contributions welcome! Here's how:

1. Fork the repo
2. Create a branch (`git checkout -b my-change`)
3. Make your changes
4. Test by running the install script in a test repo
5. Commit using [conventional commits](https://www.conventionalcommits.org/) (of course)
6. Open a PR

## What to contribute

- New config presets
- Install script improvements (new platforms, edge cases)
- Better commit range detection in CI
- Documentation fixes

## Reporting issues

Open a GitHub issue with:
- What you expected
- What happened
- Your package manager and Node.js version
